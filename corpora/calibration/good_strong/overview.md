# Helix Knowledge Base — Overview

The Helix knowledge base organizes operational documents into **catalogs**. Each catalog
holds **records**, and every record is owned by exactly one team. Terminology is consistent
across this knowledge base: catalog, record, owner.

## Structure

- **Catalog** — a named, governed set of records.
- **Record** — a single document with an owner and a source.
- **Owner** — the team accountable for a record's accuracy.
